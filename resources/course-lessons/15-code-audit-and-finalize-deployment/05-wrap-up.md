# Wrap Up

We have come to the end of the project and course. We have a working MVP and I don't think we wrote a single full line of code. Now, even though that may be true, I know and understand this entire codebase. Not every line of syntax, but I understand the complete architecture, I know the components, the actions, api routes and so on. If something breaks and for some reason, AI can't fix it, I'll know where to look.

I know that the way we did things is very different than traditional coding. Some of you may love it, some of you may hate it but either way, it remains true that AI coding isn't going anywhere and it's going to be required for most software development positions.

Now that is definitely not to say that you shouldn't learn how to write code on your own. In fact, quite the opposite. It's more important than ever to understand because you have to be the project architect.

Not even just coding itself, but if I weren't a developer, how would I even know where to add an api key as an environment variable? How would I know what database to use and how to create a dev and production branch? How would I know the little details about changes to libraries that's not in the AIs training data? You have to be connected to that world. You can't just vibe code without knowledge because you'll constantly run into barriers.

We used a strict workflow that we could reuse for every feature. Yes we had some context that the AI always knew, but aside from that, every feature was looked at as a clean slate. You don't have to use the same exact workflow that I did in this course, you could create your own and hopefully I gave you some insight on how to do that.

Also, you don't have to use 100% AI. You can still write code. Use AI for the boilerplate stuff. An example would be the Next Auth setup. That's pretty much the same for all projects. AI can take care of that for you and you can work on the soul of your project.

Continue to use AI to audit its own code and write tests. Do the things that we didn't have time to do when we would manually write code. Your productivity is 5 to 10x what it was.

As for Devstash, If you followed along and ended up with something totally different, that's fine. This is not like other courses where your end result needs to match mine. As long as it works, you're good. Keep customizing and building.

I hope you enjoyed this course and I will see you in the next one!
